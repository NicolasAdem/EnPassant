[DONE] 2. Swiss algorithm (proper rewrite).
  Files: app/services/pairing.py
  Key implementation points to verify:
    a. _backtrack worst-case baseline uses list(group), not list(top),
       so bottom-half players don't vanish.
    b. Floater cascade carries the entire unmatched remainder between
       score groups, not just one player.
    c. _max_matching does full maximum-matching search over the group
       (not just permuting the bottom half). _greedy_matching is the
       fallback for groups >14 players.
    d. After the score-group cascade, if anyone is still unmatched,
       _pair_group is retried on the full pool (step 6) before any
       emergency bye is emitted.
  Tests: test_swiss.py, check_optimality.py, test_quality.py.
  Invariant: zero rematches, zero missing players, zero emergency byes
  except when logically impossible (e.g. n=3 with 5 rounds).

[DONE] 4. Buchholz + Sonneborn-Berger tiebreaks.
  Files: app/database.py, app/services/tournament.py, templates/host.html,
         templates/projector.html, tests/test_tiebreaks.py
  - players table has `sonneborn_berger REAL NOT NULL DEFAULT 0`,
    added via idempotent _migrate helper (PRAGMA table_info guard).
  - _recompute_tiebreaks computes both tiebreaks in one pass per
    player; called from _finalize_match.
  - Definitions:
      Buchholz: sum of distinct opponents' final scores. Byes excluded.
      SB: sum over each game of (opponent_score * weight) where
          weight = 1.0 / 0.5 / 0.0 for win / draw / loss. Byes excluded.
  - list_players sorts by:
      score DESC, buchholz DESC, sonneborn_berger DESC, elo DESC, name ASC
  - UI: host.html standings has SB column next to BH; projector
    leaderboard shows "BH X · SB Y".

[DONE] 10. Rank-change ticker copy.
  Files: app/services/tournament.py, templates/projector.html,
         tests/test_rank_change.py
  - _current_ranks(conn, tid) helper returns {player_id: rank} using
    the SAME SQL ORDER BY as list_players.
  - _finalize_match snapshots ranks before the score update, snapshots
    after _recompute_tiebreaks, emits 'rank_change' events for every
    player whose rank moved. Message: "X climbed from #N to #M" or
    "X dropped from #N to #M". Payload: {player_id, name, from_rank,
    to_rank, direction}.
  - Cap: 4 rank_change events per finalize, sorted by |delta| DESC.
  - Design choice: rank changes are emitted for ALL movers, not just
    the two in the match. Buchholz cascades cause non-players to move
    when an opponent's score changes — this is correct and matches
    the leaderboard.
  - renderTicker's filter includes 'rank_change'. addTickerItem parses
    payload.direction and picks ↑/↓. Existing icons: ♟ result,
    ◆ round_start, + join.

[DONE] 11. Color balance refinement in Swiss.
  Files: app/services/pairing.py, tests/test_color_streaks.py
  - _color_streak(player_id, past_matches): walks newest to oldest,
    counts trailing non-bye same-color games. Byes are NEUTRAL (skipped,
    do not reset). Returns (color, length); (None, 0) for no history.
  - _assign_colors priority-1 streak check ABOVE the cumulative rule:
      - Exactly one player on a ≥2 same-color streak → they get the
        opposite color.
      - Both on opposite-color streaks → forced compatible (each gets
        their preferred opposite).
      - Both on same-color streaks → forced extension, falls through
        to cumulative; lower-whites player takes the extension.
  - Rules 2-4 (cumulative whites, cumulative blacks, lex id) are
    byte-identical to pre-task-11.
  - _assign_colors_rr (round-robin) is untouched — already alternates
    via _last_color.
  - pair_random uses _assign_colors → inherits fix for free.
